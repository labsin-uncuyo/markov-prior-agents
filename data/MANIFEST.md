# Data manifest

This directory contains reproducibility artifacts for the Markov-prior agents paper.

## Layout

```text
data/
├── raw/
│   ├── markov_chain/
│   │   ├── full/
│   │   ├── three_network/
│   │   └── defender/
│   └── llm/
│       ├── baseline/
│       └── mc_guided/
└── processed/
    ├── markov_chain/
    │   ├── full/
    │   ├── three_network/
    │   └── defender/
    └── llm/
```

## Markov-chain raw artifacts

The files under `data/raw/markov_chain/` are compressed preprocessed outputs from NetSecGame runs. Each file contains runs, episodes, and action-level records with reward/evaluation scores.

Scenario folders:

- `full/`: full NetSecGame scenario used for the main standalone MC-agent evaluation.
- `three_network/`: harder three-network topology used for within-NetSecGame transfer evaluation.
- `defender/`: full scenario with the stochastic defender enabled.

Filename conventions:

| File pattern | Agent | Scenario meaning |
| --- | --- | --- |
| `parsed_populationGA*.json.gz` | GA-derived Markov Chain agent | suffix determines scenario |
| `parsed_populationRA10*.json.gz` | Random-agent top-10% transition matrix | suffix determines scenario |
| `parsed_populationGPT*.json.gz` | GPT-4o-derived transition matrix | suffix determines scenario |
| `parsed_populationED*.json.gz` | Prompted expert-style / GPT o3-mini transition matrix | suffix determines scenario |
| no suffix | full scenario |
| `3` suffix | three-network scenario |
| `D` suffix | defender scenario |

For the Markov-chain experiments, each scenario/agent file contains 30 runs of 2500 episodes, i.e. 75,000 episodes per agent/scenario.

## LLM/ReAct raw artifacts

LLM/ReAct artifacts are organized by evaluation size:

- `data/raw/llm/150_episodes/`: revised 150-episode experiment reported in the manuscript.
  - `baseline/episode_data.json.gz`: baseline LLM/ReAct episodes without Markov-chain action-type guidance.
  - `baseline/llm_react.log.gz`: baseline runtime and LLM-call log.
  - `baseline/NSG_coordinator.log.gz`: NetSecGame coordinator log for the baseline run.
  - `baseline/summary.txt`: human-readable run summary.
  - `mc_guided/episode_data.json.gz`: Markov-guided LLM/ReAct episodes.
  - `mc_guided/llm_react.log.gz`: Markov-guided runtime and LLM-call log.
  - `mc_guided/summary.txt`: human-readable run summary.
- `data/raw/llm/30_episodes/`: earlier 30-episode exploratory run retained for provenance.

These files include per-episode states, prompts/responses when recorded, action evaluation scores, and end reasons. The compressed log files support the runtime and LLM-call statistics reported for the 150-episode experiment.

## Processed artifacts

Processed CSV files are generated by scripts in `scripts/analysis/` and are stored under `data/processed/`.

Markov-chain processed outputs:

- `table_metrics.csv`: run/episode metrics used to support manuscript tables.
- `terminal_outcome_summary.csv`: terminal episode outcome percentages for ALL and BEPR.
- `action_outcome_summary.csv`: action outcome percentages for ALL and BEPR.

LLM processed outputs are stored by dataset, e.g. `data/processed/llm/150_episodes/` and `data/processed/llm/30_episodes/`:

- `action_outcome_summary.csv`: invalid/redundant/effective action percentages.
- `episode_length_summary.csv`: per-episode action counts and end reasons.
- `agent_summary.csv`: compact per-agent summary, including the 150-episode success counts and step statistics.
- `runtime_summary.csv`: wall-clock time and LLM-call statistics when logs are available.

Older CSV summaries copied from the analysis repository are retained in `data/processed/markov_chain/full/` for provenance.
